## Redirect Strategy for Dropped Posts

All 87 dropped posts redirect to the blog listing in the appropriate language:

```
# HR: old root URL → /hr/blog/
/{hr-slug}/ → /hr/blog/ (301)

# EN: old /en/ URL → /blog/
/en/{en-slug}/ → /blog/ (301)

# DE: old /de/ URL → /de/blog/
/de/{de-slug}/ → /de/blog/ (301)
```

This means 87 × 3 = 261 individual redirect rules for dropped posts, generated automatically from blog frontmatter at build time.

---

## Content Directory Impact

The 87 dropped blog post directories in `content/blog/` are NOT deleted. They remain as source content (for potential future reuse) but are excluded from the build process. The build script reads the curation manifest to determine which posts to generate pages for.

**Build filter logic:**
```
for each blog directory in content/blog/:
    if slug is in KEEP list → generate /blog/[slug]/ pages (3 languages)
    if slug is in DROP list → generate next.config.ts redirect rules only
```

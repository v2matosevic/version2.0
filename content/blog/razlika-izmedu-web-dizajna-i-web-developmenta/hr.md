# Web dizajn vs. web development: Prava razlika

"Trebamo nekoga da nam dizajnira web stranicu." Devet od deset puta, ono što ljudi misle je "trebamo nekoga da nam izradi web stranicu." Koriste "dizajn" kao skraćenicu za cijeli proces. To je razumljivo. Ali ako nekoga angažirate, razlika je važna.

Web dizajn i web development su dvije različite discipline. Preklapaju se, ovise jedna o drugoj i ponekad ista osoba radi oboje. Ali zahtijevaju različite vještine, različite alate i različite načine razmišljanja.

Raščistimo ovo.

## Što web dizajn zapravo znači

Web dizajn bavi se time kako web stranica izgleda i kakav osjećaj pruža. To je vizualni sloj. Boje, tipografija, razmaci, raspored, slike, ikone. Sve ono što primijetite kad prvi put dođete na stranicu.

Ali dizajn ide dublje od estetike. Dobar web dizajner ne samo da čini stvari lijepima. On rješava probleme. Gdje bi trebala biti najvažnija informacija? Kako bi korisnik trebao navigirati od početne stranice do stranice proizvoda? Kako bi [gumbi za poziv na akciju](/blog/kako-izraditi-privlacne-cta-ove/) trebali izgledati da ih ljudi zaista kliknu?

To je [dizajn korisničkog iskustva (UX)](/blog/kako-poboljsati-korisnicko-iskustvo-ux-na-svojoj-web-stranici/). Radi se o tome da stranica bude intuitivna. Kada netko može koristiti vašu web stranicu bez razmišljanja o tome kako je koristiti, to je dobar dizajn na djelu.

Web dizajneri obično rade s alatima poput Figme, koja je postala industrijski standard za kolaborativni dizajn. Neki timovi još koriste Sketch na Macu. Izrađuju mockupove i prototipove koji pokazuju kako će gotova stranica izgledati prije nego se napiše ijedan red koda. Razmišljaju o tome [kako boje utječu na percepciju](/blog/kako-boje-utjecu-na-iskustvo-korisnika-vase-stranice/), kako tipografija utječe na čitljivost i kako vizualna hijerarhija usmjerava pažnju.

Rezultat rada web dizajnera obično je skup dizajnerskih datoteka. Piksel-savršeni rasporedi za svaki tip stranice, vodič za stil i specifikacije koje developeri koriste kao nacrt.

## Što web development zapravo znači

Web development bavi se time kako web stranica radi. To je funkcionalni sloj. Kod, baze podataka, serveri, API-jevi, performanse, sigurnost.

Developeri uzimaju dizajn i pretvaraju ga u živu, dišuću web stranicu. Pišu HTML koji strukturira sadržaj, CSS koji ga stilizira i JavaScript koji ga čini interaktivnim. Grade sustave koji vam omogućuju ažuriranje sadržaja, obradu narudžbi, slanje emailova i upravljanje korisničkim računima.

Development se dijeli na dvije specijalizacije.

**Front-end development** je kod koji se izvršava u pregledniku. Odgovoran je za pretvaranje dizajnerskih datoteka u stvarne web stranice. Svaka animacija, padajući izbornik, validacija forme i responzivni raspored je front-end posao. Front-end developeri duboko brinu o [brzini učitavanja stranice](/blog/kako-optimizirati-brzinu-web-stranice/) jer svaki kilobajt nepotrebnog koda čini stranicu sporijim.

**Back-end development** je kod koji se izvršava na serveru. Upravlja obradom podataka, autentikacijom, upravljanjem sadržajem, operacijama baze podataka i integracijama s uslugama trećih strana. Kada pošaljete kontakt formu i dobijete potvrdni email, to je back-end logika.

Jednostavna brošurna web stranica možda treba minimalan back-end rad. E-commerce stranica s upravljanjem inventarom, obradom plaćanja i korisničkim računima zahtijeva značajnu back-end arhitekturu.

## Gdje se preklapaju

Granica između dizajna i developmenta nije uvijek oštra.

CSS (koji stilizira web stranice) živi u oba svijeta. Dizajner odluči da gumbi trebaju biti plavi sa zaobljenim kutovima. Developer to implementira u CSS-u. Ali developer također odlučuje kako se ti gumbi animiraju na prelazak mišem, kako se prerasporede na malim ekranima i kako upravljaju stanjima fokusa za navigaciju tipkovnicom.

Interaktivni elementi dodatno zamagljuju granicu. Dizajner može skicirati padajući izbornik. Ali developer određuje kako se on zapravo otvara, kako upravlja interakcijom tipkovnice, što se događa na dodirnim uređajima i kako performira s 200 stavki izbornika umjesto 5.

Zato najbolji web projekti uključuju dizajnere i developere koji rade zajedno, ne jedan za drugim već paralelno. Dizajnerske odluke imaju tehničke implikacije. Tehnička ograničenja oblikuju dizajnerske mogućnosti.

## Zašto je razlika važna pri angažiranju

Ako trebate web stranicu od nule, trebate i dizajn i development. Neke tvrtke (poput nas) rade oboje. Neki freelanceri specijalizirani su za jedno ili drugo.

Evo gdje to postaje praktično.

Ako vaša postojeća stranica funkcionira dobro ali izgleda zastarjelo, primarno trebate dizajnera. Podloga koda možda je savršeno funkcionalna. Možda je potrebno samo vizualno osvježenje.

Ako vaša stranica izgleda odlično ali se učitava sporo, pada na mobitelu ili ima pokvarene funkcionalnosti, trebate developera. Dizajn nije problem.

Ako počinjete od nule, trebate puni [proces web dizajna i developmenta](/services/web-design/). Pokušaj uštede preskakanjem jedne discipline obično košta više na kraju.

## Realnost full-stacka

Vjerojatno ste čuli pojam "full-stack developer." To znači nekoga tko se bavi i front-end i back-end developmentom. Neki full-stack developeri imaju i dizajnerske vještine.

Ti ljudi postoje. Mogu izgraditi cijelu web stranicu sami. Ali postoji kompromis. Biti dobar u svemu teže je nego biti izvrstan u jednoj stvari. Za male projekte, vješt full-stack stručnjak je učinkovit i isplativ. Za veće projekte, specijalizirane uloge proizvode bolje rezultate.

Na razini studija, imati posvećene dizajnere i developere znači da svaka osoba donosi duboku stručnost u svoju domenu. Dizajner se ne ometava konfiguracijom servera. Developer ne preispituje tipografske izbore. Svatko radi ono u čemu je najbolji.

## Što to znači za vaš projekt

Ako planirate web stranicu, evo praktičnog zaključka.

Ne preskačite dizajn i ne idite ravno na development. Završit ćete s nečim što radi, ali se ne povezuje s vašom publikom. Dizajn je mjesto gdje se strategija susreće s vizualnim.

Ne zaustavljajte se na dizajnu bez pravilnog developmenta. Prekrasan mockup koji se sporo učitava, pada na mobitelu ili ima [sigurnosne ranjivosti](/blog/kako-svoju-web-stranicu-odrzati-sigurnom/) ne služi vašem poslovanju.

Obje discipline postoje s razlogom. To su različite vještine koje proizvode različite rezultate. Zajedno stvaraju web stranice koje izgledaju kako treba, rade kako treba i zaista donose rezultate.
